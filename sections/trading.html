Personal Project

Automated Crypto Trading System

GitHub →
C++ / Python · Binance API · Docker · GitHub Actions

Built from scratch to push myself into real high-performance system design. Python pulls live market data from Binance, C++ makes the trading decisions. They're isolated over a socket — if Python crashes, the C++ engine keeps running. Inside C++, three dedicated threads communicate through thread-safe queues to keep everything predictable and fast.

But the part I'm most proud of is the engineering discipline around the system — the tooling, the automation, the CI/CD gate. The goal was simple: build a system where problems get caught automatically, before they ever reach production.

Enhancement Tools & Automation

Manual logging in C++ is a nightmare — you end up cluttering source code everywhere. So I built Python injection tools that run during the build process. Zero changes to source code required.

// CppLogInjector.py

Automatically injects entry/exit timestamps into every C++ function at build time. Complete flight-recorder view of timing — no manual logging, no source changes.

BEFORE · original source
bool ExecuteBuyOrder(
  double price,
  ActionSignal signal)
{
  // business logic
  return placeOrder(price);
}
AFTER · auto-instrumented
bool ExecuteBuyOrder(
  double price,
  ActionSignal signal)
{
  LOG(CustomerLogLevel::INFO)
    << " This is start of "<< __FILE__ << "::" << __FUNCTION__;
  // Original logic
  return placeOrder(price);
  LOG(CustomerLogLevel::INFO)
    << " This is stop of "<< __FILE__ << "::"<< __FUNCTION__;
}
// Add_check_all.py

Reads a YAML config to know which functions to protect. Injects parameter validation + try-catch blocks. Uses C++17 if constexpr for compile-time type dispatch — zero runtime overhead.

Inside check_all() — the injected validation function:

⚡ if constexpr Dispatch
Single compile-time branch covers every parameter type — no runtime overhead, no specialisation boilerplate.

Integrals: positivity check
Floats: std::isfinite
Pointers: nullptr guard
Enums (ActionType): range gate
Structs (ActionSignal): recursive validation
STL Containers: non-empty guard
🔒 Fail-Fast Design
static_assert triggers a compile-time error on any unsupported type — nothing slips through silently.

Adding a new variable type = one branch in ParameterCheck.h. The Python script scales injection to hundreds of functions instantly — no manual changes to source.
BEFORE · no validation
bool ExecuteBuyOrder(
  double price,
  double amount)
{
  // no validation
  // no error handling
}
AFTER · auto-enhanced
bool ExecuteBuyOrder(
  double price,
  double amount)
{
  if(!check_all("ExecuteBuyOrder",
      price, amount)) {
    std::cerr << "Invalid parameters in "
      "TradeExecutor:: ExecuteBuyOrder!"
      "See parameter_check.log for details"
      << std::endl;
    return false;
  }
  try {
    // Original logic
  } catch(...) {
    ErrorLogger::Log(...);
    return false;
  }
}
// generate_code.py + RunTradeSystem.py

Pick a strategy in a config file. The pipeline reads it, generates the C++ wrapper and updates the makefile — only that strategy compiles in. No matter how many strategies exist in the codebase, binary size stays constant. OTA stays clean.

CI/CD Pipeline

Every commit triggers a 2-minute live run against real Binance market data. Generates trading results + performance report. A regression gate checks memory, thread count, and handle trends — merge is automatically blocked if any metric shows a statistically significant upward slope.
