Project 1 · IEC 61850 Real-Time System Transformation

This started with what looked like a small bug. Working on core algorithms for an IEC 61850 real-time system on Windows, I noticed the system was creating a new set of threads every 20ms cycle — but they were never being destroyed. Happy path tests were too short to catch it. But I knew: this thing runs 24/7 in a substation. It's only a matter of time before it crashes.

So I dug deeper. Even if I fixed the cleanup issue, was the design actually safe? I did the math:

We had 48 channels running at 50Hz

Cycle time 1000ms ÷ 50Hz = 20ms per cycle
Threads per cycle 48 (FFT) + 48÷3 (circuit analysis) = 64 threads
Threads per second (1000 ÷ 20) × 64 = 3,200 threads/sec created and destroyed

On Windows, context switching overhead alone would eat all the CPU before any real work started. I proved mathematically that the 20ms requirement was not achievable with this architecture.

So I ran a direct benchmark to make the case for Linux. Under 1,600 tasks/sec — simulating 61850 real-time load — Linux finished in 1.1s while Windows took 2.0s. A 75% gap. That data gave us the evidence we needed to migrate.

Project 2 · Linux Migration · CentOS 8 → Ubuntu

When CentOS 8 hit end-of-life, most teams treat it as a pure sysadmin task — swap the OS, test, done. I saw something different: an opportunity to fix a design that had been quietly accumulating debt for years.

The legacy stack was three layers deep — Windows services inside Wine, inside Docker, on CentOS. Every layer added overhead, added complexity, and added failure points. It was the kind of setup that works until it doesn't — and when it breaks, nobody knows where to look.

I removed Docker entirely. I stabilised Wine natively on Ubuntu — manually mapping all required Windows DLL dependencies and winetricks from scratch. Then I replaced Docker's process management with Supervisord, keeping the 24/7 self-healing behaviour but with direct kernel access and a much smaller resource footprint.

Before: Windows Service → Wine → Docker → CentOS 8  (3 layers)
After: Windows Service → Wine → Ubuntu  (2 layers)
