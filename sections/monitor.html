Real-time Performance Monitor

GitHub → Live Demo →
Python · JavaScript · Docker · Vercel · GitHub Actions

This tool came from a frustration I kept running into. In my own C++ trading system and on the IEC 61850 platform at work, some of the worst bugs are completely silent — they don't crash anything right away. Memory creeps up, thread count grows, handles never get released. Hours later the system falls over, and you have no idea why. Happy path tests are too short to catch this. They run for a few seconds and move on.

The key insight: watch the trend, not just the threshold. A slope is a problem. So I built a monitor that samples memory, thread count, handle count, and context switches every N seconds — aggregates them into trend points — then runs linear regression to detect upward slope. Integrated into CI/CD, if any metric shows a statistically significant upward trend, the pipeline automatically blocks the merge.

I also wanted it to be zero-friction for the team — there's a live web dashboard where anyone can check real-time system health just by opening a link. No setup, no SSH, no tooling.
